# Presenter Notes

## Delivery Guidance
- Repeatedly ask learners where the control signal stops.
- Differentiate low-voltage control reasoning from line-voltage motor conclusions.
- Keep safeties framed as useful diagnostic evidence, not nuisance devices.

## Suggested Opening Script
“Many HVAC troubleshooting errors begin when someone treats the BAS request as proof the unit should already be running. Today we will separate request, permission, and final command so the fault becomes obvious.”

## Key Questions to Ask During Delivery
1. What conditions must be true after the BAS call but before the fan starts?
2. Where do you expect control voltage to be present next?
3. Which device interrupts the path in this scenario?
4. What evidence rules out a failed motor as the first diagnosis?

## Common Learner Errors to Correct
- Confusing command with permissive completion
- Assuming low-voltage circuits are simple enough to guess through
- Blaming downstream hardware before tracing upstream logic

## Debrief Script
“The fan did not stay off because the BAS failed to ask. It stayed off because a safety in the control path remained open. Once again, the system logic did its job.”
