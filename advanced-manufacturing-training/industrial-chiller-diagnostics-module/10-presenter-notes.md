# Presenter Notes

## Delivery Guidance
- Keep learners focused on the difference between controller truth and actual process truth.
- Emphasize that the controller can be logically correct while the diagnosis behind it is still incomplete.
- Push learners to compare data sources, not trust a single screen blindly.

## Suggested Opening Script
“Chillers are expensive, protected machines. When they refuse to restart, the answer is usually not to press reset harder. The answer is to decide whether the machine sees a real hazard or false evidence of one.”

## Key Questions to Ask During Delivery
1. What condition is the controller trying to protect against?
2. Which data point conflicts with the rest of the evidence?
3. Is the controller wrong, or is its input wrong?
4. What must be proven before any reset is justified?

## Common Learner Errors to Correct
- Treating alarm text as the final diagnosis
- Confusing controller action with controller failure
- Skipping independent measurement of the process condition

## Debrief Script
“The controller did exactly what it was designed to do. The true fault was not the lockout itself, but the false low-temperature signal that triggered it.”
