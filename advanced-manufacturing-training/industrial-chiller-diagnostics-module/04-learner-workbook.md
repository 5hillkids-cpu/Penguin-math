# Learner Workbook

## Lesson Title
Industrial Chiller Control and Diagnostics

## Learning Goals
Use controller state, alarm history, sensor values, and process reasoning to diagnose why a chiller locks out or refuses to enable.

## Key Terms Table
| Term | Meaning in this system |
| --- | --- |
| Lockout | Protective controller state that prevents restart until a condition is cleared and reset |
| Leaving-water temperature sensor | Sensor used to monitor chilled-water temperature leaving the evaporator |
| Flow proof | Confirmation that water flow exists before compressor operation |
| Enable permissive | Required condition for compressor start |
| Alarm history | Logged sequence of controller events |

## Guided Notes
1. A controller may be making the correct decision based on ________ data.
2. Before resetting a chiller lockout, confirm whether the reported condition is ________.
3. A low leaving-water reading can come from real process conditions or a faulty ________.

## Troubleshooting Log
1. State the complaint.
2. List the first three checks.
3. Identify which condition blocked compressor enable.
4. Explain whether the controller logic was correct.
5. Record the corrective action and restart steps.

## Exit Ticket
Explain how you would prove the difference between a real low-temperature event and a false low-temperature sensor signal.
